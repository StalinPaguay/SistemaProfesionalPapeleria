Documentación Oficial: PaperDS (Backend)

Bienvenido a la documentación arquitectónica y técnica del Sistema Profesional de Papelería (PaperDS). Esta plataforma ha sido diseñada bajo estrictos estándares de la industria del software utilizando el framework Spring Boot 3.

1. Arquitectura del Sistema

El sistema está fundamentado en una Arquitectura Multicapa (N-Tier) orientada a dominios, garantizando bajo acoplamiento y alta cohesión. Las capas principales son:

  • Controladores (REST API): Endpoints seguros y documentados que sirven como interfaz de comunicación con el Frontend (Angular).
  • Servicios (Business Logic): Capa transaccional (@Transactional) donde residen las reglas de negocio, validaciones de stock y cálculos financieros.
  • Repositorios (Data Access): Integración fluida con PostgreSQL utilizando Spring Data JPA y consultas JPQL optimizadas.
  • DTOs y Mappers: Objetos de Transferencia de Datos que actúan como barrera de seguridad para no exponer las entidades físicas directamente a la web.

2. Seguridad (DevSecOps)

PaperDS implementa un robusto esquema de seguridad sin estado basado en JSON Web Tokens (JWT) integrado con Spring Security. Las rutas están aseguradas por un control de acceso basado en roles (RBAC), separando estrictamente a los USUARIOS de los ADMINISTRADORES.
